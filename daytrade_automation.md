## morning_screening.py 処理フロー

### STEP 0：買付余力取得（kabuステーションAPI）
```
GET /wallet/cash
→ 買付余力 × INVESTMENT_RATIO（0.80）= 本日の投資予算
→ API取得失敗時は前日レポートJSONのclosing_walletを使用
→ それも取得できない場合は固定予算800,000円を使用
```

### STEP 1a：出来高急増スクリーニング（J-Quants API）

screener.py の get_volume_surge_candidates() で以下のフィルタを順次適用：

| フィルタ | 条件 |
|---------|------|
| [1/7] 対象日フィルタ | 前営業日のデータを使用 |
| [2/7] 出来高急増 | 直近20日平均の2倍以上 |
| [3/7] 売買代金 | 5日平均3,000万円以上 |
| [4/7] 4桁コード | 通常株式のみ（ETF・ETN・REIT除外） |
| [4.5/7] ETF除外 | 上場銘柄マスタで分類確認 |
| [4.6/7] ボラティリティ | 前日の日中値幅1.5%以上 |
| [4.7/7] 前日急騰除外 | 前日終値が前々日比+5%以上 → 除外（3/18追加） |
| [5/7] 時価総額 | 50億円以上 |
| [6/7] 赤字銘柄除外 | 未実装（Claude材料判定で代替） |
| [7/7] 予算フィルタ | 1単元（100株×終値）≤ 投資予算 |

**前日急騰除外フィルタの根拠：**
バックテスト分析（240営業日）で、前日+5%以上の銘柄は勝率14.3%・平均損益-5.60円と判明。
+5%未満の銘柄（勝率43.1%・平均+4.01円）と比較して著しく劣後するため除外。
この1フィルタで合計損益+45%改善・最大DD半減を確認済み。

### STEP 1b：TDnet引け後適時開示取得
```
前営業日15:30以降の開示を全件取得
→ 出来高急増リストに未含有の銘柄のみ、J-QuantsでOHLCV取得してconcat
→ 上限：最大候補数20銘柄
```

### STEP 2：地合いチェック（kabuStation先物API優先）
```
メイン指標: kabuステーションAPI 大証日経225先物（リアルタイム）
  → /symbolname/future で直近限月の銘柄コード取得
  → /board で現在値・前日終値を取得
  → 前日比 -1.5%以上下落 → skip_all（全スキップ・Discord通知のみ）

フォールバック（kabuStation取得失敗時）:
  yfinance NKD=F（シカゴ先物）→ ^N225（日経平均指数）
  → NYダウ・ナスダックの両方が -1.5%以上下落 → skip_all

※ yfinanceは前日終値までしか反映されず夜間の回復が検知できないため
  kabuStation APIを最優先で使用（4/3修正）

Discord通知例:
  「地合い: 正常 / 日経先物 +1.3%」
  「地合い: ⚠️スキップ / 日経先物 -2.1%（閾値-1.5%超え）」
```

### STEP 3：株探ニュース取得
```
上位20銘柄について株探の銘柄ニュースをスクレイピング
→ 前営業日15:30以降のニュースのみ（鮮度フィルタ）
→ 採用カテゴリ：開示・決算・5%・材料
→ 最大3件/銘柄
```

### STEP 4：Claude API材料判定（MaterialJudge）
```python
# 各銘柄にClaude APIを呼び出してJSON形式で取得
{
  "has_material": true/false,
  "material_type": "業績上方修正|決算好調|株式分割|自社株買い|テーマ株|その他",
  "strength": "強|中|弱",
  "summary": "材料の内容（20字以内）",
  "risk": "注意点（翌日決算・高信用倍率等）またはnull"
}

除外条件：
  - has_material=false かつ strength=弱
  - 減益・赤字・下方修正・業績悪化キーワードを含む場合
  - 銘柄コードを明示して他銘柄との誤紐付けを防止
```

### STEP 4.5：前日ストップ高除外フィルタ
```
前日終値が前々日比+25%以上 → 除外
（trade_executor.py でも二重チェック済み）
```

### STEP 5：CSV保存
```
data/candidates_YYYYMMDD.csv
カラム：Code・銘柄名・終値・VolumeSurgeRatio・has_material・material_strength・material_type・material_summary
```

### STEP 6：Discord通知
```
朝レポート送信：
  - 本日の投資予算
  - 地合い（日経先物 / フォールバック時はNYダウ・ナスダック）
  - 最終候補銘柄一覧（強度・出来高倍率・材料サマリー）
  - TDnet開示件数
```

## auto_trading.py 処理フロー

### 8:45 起動処理
```
1. 営業日チェック（jpholiday）
2. kabuステーション疎通確認（GET /token）
3. candidates_YYYYMMDD.csv 確認
4. 買付余力取得 × INVESTMENT_RATIO(0.80) = 本日予算
5. execute_daily_trading() 実行
```

### execute_daily_trading()
```
1. 安全装置チェック（check_safety_conditions）
2. CSV読み込み・フィルタ適用
   - material_strength '強'または'中'のみ
   - 前日ストップ高銘柄を除外
3. 優先順位ソート
   - 強 > 中、同一強度内はVolumeSurgeRatio降順
4. 9:05以降はパターンAスキップ（寄り付き後の成行リスク回避）
5. 最優先の1銘柄に成行注文を8:45に発注（信用デイトレード・SOR）
6. 9:00:30まで待機後、5秒×12回ポーリングで約定確認
7. 実際の約定価格を取得し、損切(-1%)・利確(+2%)を設定
8. 損切り・利確はmonitor_positionsの監視型（3秒ポーリング）
9. Discord通知（エントリー銘柄・価格・損切ライン）
```

### 取引監視ループ（9:00〜15:30）
```
メインループ（60秒ごと）:
  9:30〜15:25: パターンBスキャン・エントリー
  11:25〜11:35: force_exit_losing_positions_midday()
    → 含み損ポジションを成行決済
  15:15〜15:25: force_exit_all_positions_eod()
    → 全ポジションを成行決済

ポジション監視スレッド（3秒ごと・別スレッド）:
  monitor_positions()
    → 利確判定: 現在値 >= target_price → 成行売り
    → 損切判定: 現在値 <= stop_price → 成行売り
    → MFE/MAE更新

15:30: ループ終了・日次レポート生成・Discord通知
```

## エントリータイミング

### パターンA（本番稼働中）
```
対象：前日スクリーニングで候補リスト入り済みの銘柄（材料「強」「中」のみ）
タイミング：8:45に成行注文発注 → 9:00寄り付きで約定
取引種別：信用デイトレード（CashMargin=2, MarginTradeType=3, SOR）
約定確認：9:00:30から5秒×12回ポーリング、未約定なら注文キャンセル
利確：+2.0%（monitor_positions監視型・3秒ポーリング）
損切：-1.0%（monitor_positions監視型・3秒ポーリング）
最終期限：9:05以降はスキップ
```

### パターンB（本番稼働中）
```
状態：有効化済み・本番稼働中

対象：売買高急増ランキング＋candidates CSV材料銘柄
タイミング：9:30〜15:25（取引監視ループ内で1分ごとスキャン）
取引種別：信用デイトレード（CashMargin=2, MarginTradeType=3, SOR）

スキャンフィルター（ランキング50件→最大10件に絞り込み）：
  1. ETFフィルター（銘柄名・コード・正式名で判定）
  2. 株価200円以上
  3. 予算超過除外（budget以内）
  4. 売買代金3,000万円以上
  5. /board取得後、ETF再チェック（正式名で判定）
  6. 時価総額50億円以上（fins_summaryから発行済株式数取得）
  7. 板の厚さフィルター（AskQtyがエントリー予定株数の2倍以上）
  8. 寄りから+3%以内（高値掴み防止）
  9. candidates CSVの材料銘柄（強・中）を先頭に優先追加

エントリー条件（全て満たす場合）：
  1. VWAPより上（0.2%バッファあり）
  2. 直近5本中3本以上が上昇トレンド
  3. 出来高急増（RapidTradePercentage 100%以上）
  4. active_positionsが0件

発注：指値（現在値+1%）+ 5秒×6回約定確認
利確：+1.5%（MFE分析: 14トレードで+2%到達0件のため変更）
損切：-1.0%（monitor_positions監視型・3秒ポーリング）
```

## 注意事項

1. **信用取引（デイトレード）** ― kabuステーションAPI経由の信用取引は手数料・金利・貸株料すべて無料
2. **kabuステーション起動必須** ― APIはkabuステーション起動中のみ有効
3. **監視型損切り** ― 逆指値APIではなく3秒ポーリングで監視型決済（SOR+信用で逆指値API非対応のため）
4. **SOR注文** ― Exchange=9（SOR）で発注。Exchange=1（東証）は当口座で利用不可
